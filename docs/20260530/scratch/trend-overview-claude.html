4. 기술 원리 / 해결 문제 / 트레이드오프 Matrix

기술원리해결 문제Backend Risk적용 기준
LLM API 대규모 사전학습 모델에 prompt를 전달해 token 생성 자연어 이해, 코드 생성, 문서 요약, 분류 자동화 비결정성, hallucination, vendor lock-in, token cost 업무 보조, 초안 생성, 비정형 텍스트 처리
RAG 질문 → embedding 검색 → 관련 문서 context 주입 → 생성 최신/사내 지식 반영, 출처 기반 답변, fine-tuning 비용 절감 chunking 실패, stale index, 권한 필터 누락, retrieval latency 사내 문서 QA, 정책 검색, 고객지원, 기술문서 탐색
VectorDB 고차원 embedding에 대한 approximate nearest neighbor 검색 키워드 검색이 약한 의미 기반 탐색 index rebuild 비용, metadata filter 성능, multi-tenant isolation 문서량이 많고 의미 검색이 필요한 경우
Function Calling / Tool Use 모델이 JSON schema에 맞춰 외부 함수 호출 의도를 생성 DB 조회, API 호출, 계산, workflow 실행 연결 권한 상승, prompt injection, idempotency 실패, side effect 통제 읽기 작업부터 시작, 쓰기 작업은 human approval 권장
Agentic Workflow 계획 → 도구 호출 → 관찰 → 재시도 → 결과 생성 loop 단발성 prompt로 어려운 다단계 업무 자동화 무한 루프, 비용 폭증, 잘못된 tool chaining, audit 어려움 명확한 종료조건, budget, sandbox, trace 저장이 있을 때
SLM Routing 작은 모델로 분류/초안/간단 작업 처리, 어려운 작업만 frontier model로 escalte 비용과 latency 절감 잘못된 triage, 품질 편차, eval set 부족 대량 요청, 반복 업무, 난이도 분포가 넓은 서비스
On-Device AI 모델 일부/전체를 클라이언트 장치에서 실행 개인정보 보호, 오프라인 처리, 서버 비용 절감 기기별 성능 편차, 모델 업데이트 어려움, 보안 검증 한계 민감 데이터 전처리, 요약, 키보드/음성/이미지 보조
Harness / Eval Engineering 프롬프트, 모델, retrieval, tool 결과를 fixture와 metric으로 검증 환각 감소, 회귀 테스트, 모델 교체 안정성 확보 테스트셋 오염, metric과 실제 품질 불일치 운영 전 필수. 특히 금융/정산/권한/보안 도메인

5. Backend 적용 판단 기준

적용해도 되는 경우

  • 정답이 아니라 후보/초안/분류가 필요한 경우
  • 결과를 검증할 rule, schema, human approval이 있는 경우
  • 실패해도 rollback 가능한 읽기 중심 workflow
  • 모델 교체를 대비한 adapter, gateway, eval set이 있는 경우

보류해야 하는 경우

  • 법적/금전적 side effect를 자동 실행해야 하는 경우
  • 권한별 데이터 격리가 retrieval 단계에서 보장되지 않는 경우
  • latency SLO가 짧고 cold start/API dependency가 큰 경우
  • 관측성 없이 agent가 외부 tool을 연쇄 호출하는 경우

6. 2026 이후 전망

Model단일 최고 모델보다 task-specific routing, small/frontier hybrid, reasoning budget 제어가 중요해진다.
ArchitectureLLM은 서비스 내부 모듈이 아니라 외부 불확실성을 가진 dependency로 취급해야 한다.
SecurityPrompt injection, tool abuse, data exfiltration, model supply chain 검증이 표준 보안 항목이 된다.
CostCache, batch, SLM triage, embedding 재사용, context pruning, async job queue가 비용 최적화 핵심이다.
Backend Role백엔드 엔지니어의 역할은 “AI 호출”이 아니라 workflow boundary, data contract, observability, rollback 설계다.
